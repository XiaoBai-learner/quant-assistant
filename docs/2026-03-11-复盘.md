## 一、架构层面分析

### 1.1 现有架构的优点

| 方面 | 评价 |
|------|------|
| 模块化设计 | ✅ 分层清晰，职责分离 |
| 事件驱动 | ✅ 核心模块解耦，便于扩展 |
| 单例模式 | ✅ EventBus/GlobalContext 确保全局一致性 |
| 抽象基类 | ✅ BaseStrategy/BaseDataFetcher 便于定制 |

### 1.2 架构缺陷与修复方案

#### 缺陷 1: 缺乏依赖注入容器

**问题**: 依赖硬编码，不利于测试和扩展
```python
# 当前做法 - 硬编码依赖
class BacktestEngine:
    def __init__(self, config):
        self.broker = Broker(config)  # 硬编码
        self.portfolio = Portfolio(config.initial_cash)  # 硬编码
```

**修复方案**: 引入依赖注入
```python
class BacktestEngine:
    def __init__(self, config, broker: IBroker, portfolio: IPortfolio):
        self.broker = broker
        self.portfolio = portfolio
```

#### 缺陷 2: 配置管理混乱

**问题**: 多处配置来源不统一
- `.env` 环境变量
- `config/*.yaml` 配置文件
- `src/config.py` Python配置
- `src/utils/config.py` 另一个配置

**修复方案**: 统一配置中心
```python
# src/config_manager.py
@dataclass
class Config:
    database: DatabaseConfig
    data: DataConfig
    logging: LoggingConfig
    
    @classmethod
    def from_yaml(cls, path: str) -> "Config":
        with open(path) as f:
            data = yaml.safe_load(f)
        return cls(**data)
```

#### 缺陷 3: 缺乏接口定义

**问题**: 没有 protocol 或 abstract base class 定义清晰的接口

**修复方案**: 使用 Protocol 定义接口
```python
from typing import Protocol

class IBroker(Protocol):
    def submit_order(self, order: Order) -> bool: ...
    def execute_order(self, order: Order, bar) -> List[Trade]: ...
```

## 二、数据管理层缺陷与修复

### 2.1 严重缺陷

#### 缺陷 1: 无数据质量校验

**问题**: 直接存储，无数据校验
```python
# 当前 - 直接存储
def save_daily_quotes(self, df: pd.DataFrame) -> int:
    records = df.to_dict('records')
    # ... 直接存储
```

**修复方案**: 添加数据质量检查
```python
class DataValidator:
    def validate_price_data(self, df: pd.DataFrame) -> ValidationResult:
        checks = [
            self._check_missing_values(df),
            self._check_price_logic(df),  # high >= low >= 0
            self._check_volume_positive(df),
            self._check_duplicates(df),
            self._check_outliers(df)
        ]
        return ValidationResult(checks)
```

#### 缺陷 2: 无增量更新机制

**问题**: 每次全量更新，效率低

**修复方案**:
```python
def update_daily_data(self, symbol: str):
    last_date = storage.get_last_update_date(symbol)
    if last_date and last_date >= yesterday:
        return  # 已是最新
    
    df = fetcher.get_daily_quotes(symbol, last_date, yesterday)
    storage.save_daily_quotes(df)  # 仅增量
```

#### 缺陷 3: 缺乏数据缓存层

**问题**: 每次查询都访问数据库

**修复方案**: 添加 Redis 缓存
```python
class DataCache:
    def __init__(self, redis_client: redis.Redis):
        self.redis = redis_client
        self.ttl = 300  # 5分钟
    
    def get(self, key: str) -> Optional[pd.DataFrame]:
        data = self.redis.get(key)
        return pickle.loads(data) if data else None
```

## 三、策略层缺陷与修复

### 3.1 严重缺陷

#### 缺陷 1: 策略无法组合

**问题**: 单策略执行，无法多策略组合

**修复方案**:
```python
class CompositeStrategy(BaseStrategy):
    def __init__(self, strategies: List[BaseStrategy], weights: List[float]):
        self.strategies = strategies
        self.weights = weights
    
    def on_bar(self, bar):
        signals = [s.on_bar(bar) for s in self.strategies]
        return self._combine_signals(signals, self.weights)
```

#### 缺陷 2: 缺乏策略参数优化

**问题**: 无网格搜索、遗传算法、贝叶斯优化

**修复方案**: 使用 Optuna 优化
```python
class StrategyOptimizer:
    def optimize(self, data: pd.DataFrame, 
                 objective: Callable, n_trials: int = 100) -> Dict:
        def objective_fn(trial):
            params = {
                name: trial.suggest_float(name, low, high)
                for name, (low, high) in self.param_space.items()
            }
            strategy = self.strategy_class(**params)
            return objective(strategy, data)
        
        study = optuna.create_study(direction='maximize')
        study.optimize(objective_fn, n_trials=n_trials)
        return study.best_params
```

## 四、回测层缺陷与修复

### 4.1 严重缺陷

#### 缺陷 1: 撮合机制过于简化

**问题**: 市价单直接以 close 成交，不真实

**修复方案**: 添加订单簿模拟
```python
class RealisticBroker(Broker):
    def execute_order(self, order: Order, bar: Bar, depth: OrderBook) -> List[Trade]:
        if order.order_type == OrderType.MARKET:
            return self._match_market_order(order, depth)
        elif order.order_type == OrderType.LIMIT:
            return self._match_limit_order(order, depth)
```

#### 缺陷 2: 缺乏风控检查

**问题**: 无事前风控

**修复方案**:
```python
class RiskManager:
    def check_order(self, order: Order, portfolio: Portfolio) -> RiskCheckResult:
        checks = [
            self._check_position_limit(order, portfolio),
            self._check_cash_sufficient(order, portfolio),
            self._check_daily_loss_limit(portfolio),
        ]
        return RiskCheckResult(checks)
```

#### 缺陷 3: 绩效指标不全

**修复方案**: 添加更多指标
```python
class PerformanceMetrics:
    def calculate_all(self, returns: pd.Series) -> Dict[str, float]:
        return {
            'sharpe_ratio': self.sharpe_ratio(returns),
            'sortino_ratio': self.sortino_ratio(returns),
            'calmar_ratio': self.calmar_ratio(returns),
            'max_drawdown_duration': self.max_drawdown_duration(returns),
            'information_ratio': self.information_ratio(returns, benchmark),
        }
```
